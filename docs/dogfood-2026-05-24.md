## 2 — Boot sequence

### Option A — Single command (preferred)

```bash
bun run dev:complete
```

Brings up apps/web + apps/api + apps/ponder + all keepers + the
matcher in one process tree. Interleaved logs via Turbo. Stop with
`Ctrl-C` to kill everything.

### Option B — Three terminals (faster iteration when debugging one service)

```bash
# Terminal 1 — web on :3001
bun run dev

# Terminal 2 — api on :3002
bun run dev:api

# Terminal 3 — matcher (gRPC :3005, HTTP :3006)
bun run matcher:dev
```

The matcher reads `.env.local` from the repo root via bun's
auto-source. Direct `cargo run` requires `set -a && source
../../.env.local && set +a` first.

### Option C — Just the bare minimum for UI navigation

```bash
bun run dev          # web only — covers visual + button-click QA
                     # API + matcher offline; reads will fail gracefully
```

## 4 — Trade-to-fill walkthrough

Goal: connect a wallet, open a position, observe the matcher
settle the fill, see the position appear in the UI + a Trade message
on Redis.

### 4.1 Get past the alpha gate

The web app gates everything behind `/alpha`:

```bash
# Programmatically (curl)
PWD=$(grep '^ALPHA_GATE_PASSWORD=' .env.local | cut -d= -f2)
curl -c /tmp/cj -d "password=$PWD" http://127.0.0.1:3001/alpha?next=%2F

# In /browse
goto http://127.0.0.1:3001/
press Enter         # opens the password prompt
type "$ALPHA_GATE_PASSWORD"
press Enter
```

### 4.2 Connect wallet

Trade Island lives on `/` (the homepage), not `/trade`.

1. Click **Log in or sign up** (Dynamic widget).
2. Pick the wallet flow that matches the EOA you funded on Arc.
3. Ensure the network switches to **Arc Testnet (5042002)** — the
   UI auto-prompts the wallet to switch.

### 4.3 Open a position

In the Trade tab:

1. Pick a market — **EUR/USD** is the canary market (lowest LLTV friction).
2. Pick **Market** order type + **Long** side.
3. Set notional to the minimum (~1 USDC) to stay under the
   per-fill cap during testing.
4. Hit **Submit**.

The submit hook fires:

```
UI signs SignedOrder (EIP-712, 9 fields incl. maxFee)
    → POST /perps/intents (apps/api)
    → INSERT into perp_order_intents (sqlite)
    → matcher tick picks it up next loop (≤ 1s busy / 30s idle)
    → match against LP backstop OR rest on book
    → if LP matched: settleMatch on Arc → fill confirmed
    → trade fan-out: gRPC StreamTrades + Redis bufi:trades:<market>
    → UI's StreamTrades subscriber updates Positions tab
```

### 4.4 Observe the fill

In `/browse`:

```bash
# Snapshot UI before submit
snapshot -i -o /tmp/before.png

# Submit the order via clicks
click @e<submit-button-ref>

# Wait + snapshot after the matcher tick
wait --networkidle
snapshot -i -o /tmp/after.png

# Diff
snapshot -D
```

Expected diff: **Positions tab badge increments to 1**, the Trade
ticker shows a new fill row, the Positions tab now shows the open
position with entry price + PnL.

In a separate shell with `redis-cli psubscribe "bufi:*"` running:

```
pmessage  bufi:*  bufi:trades:565a6e2f…  <117-byte proto Trade>
pmessage  bufi:*  bufi:trades              <same bytes>
```

### 4.5 Close the position

In Positions tab → click **Close** on the row. Same flow in
reverse: signs a reduce-only order, matcher settles, position
disappears from UI.

## 5 — Known operational signals (expected warnings, not bugs)

| Log line | Meaning | Action |
|---|---|---|
| `LP router denied residual; FillSizeCap` | Per-market max-OI gate fired on a too-large canary intent. | Lower `CANARY_NOTIONAL_USDC_E6` or raise the cap with `docs/operator-raise-oi-caps.md`. |
| `LP router denied residual; OracleStale` | Pyth feed > `pyth_push_max_age` seconds old. | Check `pyth_pusher` is running + Hermes WS connected (look for "WS connected" or HTTP polling logs). |
| `pyth_pusher: feed unresolved` | One Pyth feed id not configured on the FxOracle. | Either add to FxOracle via owner tx, or accept that one market won't get fresh marks. |
| `funding: seed failed` at boot | Funding poker tried to read state before the first RPC connect. | Auto-recovers on next tick — non-fatal. |

## 6 — Hard signals to watch (real problems)

| Symptom | Likely cause |
|---|---|
| `/ready` returns 503 with `tick:false` | Matcher tick loop has stalled. Check Arc RPC reachability + DB lock. |
| Same intent retries forever with `Permit2: invalid signature` | EIP-712 typehash drift between UI + matcher. Verify `SIGNED_ORDER_TYPES` matches `matcher-types/src/eip712.rs::SignedOrder`. |
| `boot fails: key collision` | KEEPER / LP_OPERATOR / CANARY share an EOA. They MUST be distinct. |
| Frontend says "wallet not connected" after MetaMask connect | Dynamic widget on Arc Testnet has a known flake — refresh the page. |
| `bufi:trades:*` channel stays silent across multiple submits | Matcher's `realtime` task crashed. Check log for `realtime publisher exited`. |
